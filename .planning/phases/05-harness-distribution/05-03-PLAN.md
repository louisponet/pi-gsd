<interfaces>
pi-gsd-settings.json schema (JSON Schema draft-07):
  {
    "$schema": "http://json-schema.org/draft-07/schema#",
    "title": "pi-gsd Settings",
    "type": "object",
    "properties": {
      "trustedPaths": { "type": "array", "items": { "type": "string" } },
      "untrustedPaths": { "type": "array", "items": { "type": "string" } },
      "shellAllowlist": { "type": "array", "items": { "type": "string" } },
      "shellBanlist": { "type": "array", "items": { "type": "string" } },
      "shellTimeoutMs": { "type": "number", "minimum": 1000, "maximum": 300000 }
    },
    "additionalProperties": false
  }

Settings merge order (project overrides global):
  1. Start with DEFAULT_SHELL_ALLOWLIST as base allowlist.
  2. Load ~/.gsd/pi-gsd-settings.json (global). If not found → empty object.
  3. Load <cwd>/.pi/gsd/pi-gsd-settings.json (project). If not found → empty object.
  4. Merge: project values override global values for scalar fields.
     For array fields (shellAllowlist): project array REPLACES global (not appends).
  5. Apply shellBanlist: remove any banlist entries from the effective allowlist.

Effective WxpSecurityConfig:
  trustedPaths: [pkgHarness, projectHarness, ...(merged.trustedPaths ?? [])]
  shellAllowlist: merged.shellAllowlist ?? DEFAULT_SHELL_ALLOWLIST (minus banlist)
  shellTimeoutMs: merged.shellTimeoutMs ?? 30_000

The banlist takes priority: a command in the banlist is NEVER allowed, even if it's in the allowlist.

    Add to both extension files a `loadGsdSettings` function and update the context event handler:

    ```typescript
    interface GsdSettings {
      trustedPaths?: string[];
      untrustedPaths?: string[];
      shellAllowlist?: string[];
      shellBanlist?: string[];
      shellTimeoutMs?: number;
    }

    function loadGsdSettings(cwd: string): GsdSettings {
      const globalPath = join(homedir(), ".gsd", "pi-gsd-settings.json");
      const projectPath = join(cwd, ".pi", "gsd", "pi-gsd-settings.json");

      function readSettings(p: string): GsdSettings {
        try {
          if (!existsSync(p)) return {};
          return JSON.parse(readFileSync(p, "utf8")) as GsdSettings;
        } catch { return {}; }
      }

      const global = readSettings(globalPath);
      const project = readSettings(projectPath);

      // Project overrides global for all fields
      return {
        trustedPaths: project.trustedPaths ?? global.trustedPaths,
        untrustedPaths: project.untrustedPaths ?? global.untrustedPaths,
        shellAllowlist: project.shellAllowlist ?? global.shellAllowlist,
        shellBanlist: project.shellBanlist ?? global.shellBanlist,
        shellTimeoutMs: project.shellTimeoutMs ?? global.shellTimeoutMs,
      };
    }
    ```

    Update the context event's WxpSecurityConfig construction:
    ```typescript
    const userSettings = loadGsdSettings(ctx.cwd);

    // Effective allowlist: user setting or default, minus banlist
    const baseAllowlist = userSettings.shellAllowlist ?? [...DEFAULT_SHELL_ALLOWLIST];
    const banlist = userSettings.shellBanlist ?? [];
    const effectiveAllowlist = baseAllowlist.filter((cmd) => !banlist.includes(cmd));

    const security: WxpSecurityConfig = {
      trustedPaths: [
        pkgHarness,
        projectHarness,
        ...(userSettings.trustedPaths ?? []),
      ].filter(existsSync),
      shellAllowlist: effectiveAllowlist,
      shellTimeoutMs: userSettings.shellTimeoutMs ?? 30_000,
    };
    ```

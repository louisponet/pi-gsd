<interfaces>
WXP conversion pattern for workflow files:

BEFORE (LLM executes bash):
  ```
  Run: pi-gsd-tools init execute-phase $PHASE
  ```
  (The LLM would see this instruction and call a bash tool)

AFTER (WXP pre-injects the data):
  ```xml
  <gsd-arguments>
    <positional name="PHASE" />
    <flag name="auto" boolean />
    <flag name="no-transition" boolean />
  </gsd-arguments>

  <gsd-execute>
    <shell command="pi-gsd-tools" result="INIT">init execute-phase <gsd-paste name="PHASE" /></shell>
    <shell command="pi-gsd-tools" result="STATE">state json --raw</shell>
    <shell command="pi-gsd-tools" result="ROADMAP_SECTION">roadmap get-phase <gsd-paste name="PHASE" /> --pick section</shell>
  </gsd-execute>
  ```

  Then later in the document:
  ```
  ## Current State
  <gsd-paste name="STATE" />

  ## Phase Roadmap
  <gsd-paste name="ROADMAP_SECTION" />
  ```

Version tag (WFL-04): place at the very top of the file:
  ```xml
  <gsd-version v="1.12.4" />
  ```

Backup procedure:
  git cp execute-phase.md execute-phase.md.bak
  git add execute-phase.md.bak
  git commit -m "chore(workflows): backup execute-phase.md before WXP conversion"
</interfaces>
</context>

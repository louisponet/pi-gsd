<interfaces>
For each workflow, the conversion pattern is:
1. Read the workflow to identify all "Run: pi-gsd-tools ..." initialization steps.
2. Replace with WXP <gsd-arguments> + <gsd-execute>/<shell> blocks.
3. Add <gsd-paste> tags where the data should appear.
4. Add <gsd-version v="1.12.4" /> at the top.

Common data fetched by high-traffic workflows:
  plan-phase:
    - init plan-phase <PHASE> → INIT_DATA (phase info, config, paths)
    - state json --raw → STATE
    - roadmap get-phase <PHASE> --pick section → ROADMAP_SECTION

  discuss-phase:
    - init discuss-phase <PHASE> → INIT_DATA
    - state json --raw → STATE

  new-project:
    - No phase argument - minimal data needed
    - config-get workflow.discuss_mode → DISCUSS_MODE (conditional)

  new-milestone:
    - state json --raw → STATE

Each workflow file must not require the LLM to issue any bash calls for setup.
</interfaces>

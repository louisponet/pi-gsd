<interfaces>
Pi extension API — context event:
  pi.on("context", async (event, ctx) => { ... })
  event.content: string — the raw file content being added to context
  event.filePath: string — absolute path to the file
  return value: { content: string } — modified content, OR undefined (no change)

Trusted paths (runtime computation):
  1. Package harness: dirname(__filename) + "/../../.gsd/harnesses/pi/get-shit-done"
     (the extension file lives at <pkg>/.gsd/extensions/pi-gsd-hooks.ts)
  2. Project harness: ctx.cwd + "/.pi/gsd"

The context event must:
  1. Check if event.filePath is in a trusted path. If not → return undefined (pass through).
  2. Check if content contains <gsd- tags. If not → return undefined (no WXP needed).
  3. Call processWxp(content, filePath, config).
  4. On success → return { content: processedContent }.
  5. On WxpProcessingError → ctx.ui.notify(error.message, "error") and return { content: "" }
     (empty string prevents partial content from reaching LLM — total crash per WXP-09).

Note: The <gsd-include> resolution that already exists in the extension runs BEFORE the context
event's WXP processing — the order is: include resolution → WXP → LLM.
</interfaces>
</context>

    Inside the default export function (after the existing session_start, tool_call, tool_result handlers), add:

    ```typescript
    // ── context: WXP preprocessing ────────────────────────────────────────────
    pi.on("context", async (event, ctx) => {
      try {
        // Only process files that contain WXP tags
        if (!event.content.includes("<gsd-")) return undefined;

        // Compute trusted paths at runtime
        const extDir = typeof __filename !== "undefined" ? dirname(__filename) : "";
        const pkgHarness = join(extDir, "..", "..", ".gsd", "harnesses", "pi", "get-shit-done");
        const projectHarness = join(ctx.cwd, ".pi", "gsd");

        const security: WxpSecurityConfig = {
          trustedPaths: [pkgHarness, projectHarness].filter(existsSync),
          shellAllowlist: [...DEFAULT_SHELL_ALLOWLIST],
          shellTimeoutMs: 30_000,
        };

        // Read rawArguments from event metadata if available
        const rawArguments = (event as { arguments?: string }).arguments ?? "";

        const processed = processWxp(event.content, event.filePath, { security, rawArguments });
        return { content: processed };
      } catch (err) {
        if (err instanceof WxpProcessingError) {
          ctx.ui.notify(err.message, "error");
          return { content: "" }; // prevent partial content reaching LLM
        }
        // Unexpected extension error — log but don't block
        const msg = err instanceof Error ? err.message : String(err);
        ctx.ui.notify(`GSD WXP: unexpected error in context handler: ${msg}`, "info");
        return undefined;
      }
    });
    ```

    Add the necessary imports at the top of both extension files:
    ```typescript

    import { DEFAULT_SHELL_ALLOWLIST } from "../../../src/wxp/security.js";
    import type { WxpSecurityConfig } from "../../../src/wxp/schema.js";
    ```

    Note: For the distributed package, the import path will be from the compiled dist; for development it uses src directly. The build step in tsup handles this. Both files must be updated identically (they are the same content — .gsd is canonical source, .pi is project copy).
